Git/GitHub

Git एक वर्जन कंट्रोल सिस्टम है जो आपको अपने प्रोजेक्ट के इतिहास को ट्रैक करने और विभिन्न संस्करणों के साथ काम करने की अनुमति देता है। GitHub एक ऑनलाइन प्लेटफ़ॉर्म है जो Git रिपॉजिटरीज़ को होस्ट करता है और सहयोग को आसान बनाता है।

git init (एक नई Git रिपॉजिटरी बनाने के लिए)

git add . (सभी फाइलों को स्टेज करने के लिए)

git commit -m "Initial commit" (पहली कमिट बनाने के लिए)

git branch -M main (ब्रांच का नाम बदलने के लिए, जैसे कि "master" से "main" में)

git remote add origin https://github.com/USERNAME/REPO_NAME.git (रिमोट रिपॉजिटरी जोड़ने के लिए, USERNAME और REPO_NAME को अपने GitHub यूजरनेम और रिपॉजिटरी नाम से बदलें)

git remote add origin https://github.com/RSJK2562/Git-GitHub.git (रिमोट रिपॉजिटरी जोड़ने के लिए, USERNAME और REPO_NAME को अपने GitHub यूजरनेम और रिपॉजिटरी नाम से बदलें)

git push -u origin main (रिमोट रिपॉजिटरी पर पुश करने के लिए, "main" ब्रांच को ट्रैक करने के लिए)

अपनी लोकल ब्रांच का नाम "main" में बदलें और फिर पुश करें:
1. git branch -m master main
2. git push -u origin main

अगर आपने GitHub पर README, लाइसेंस, या .gitignore के साथ एक खाली रिपॉजिटरी बनाई है, तो आपको पहले उन परिवर्तनों को पुल करना होगा:
git pull origin main --allow-unrelated-histories

git remote -v (to check the remote repositories)

git remote remove origin (to remove the remote repository)

git branch (to see the current branch)

git branch -a (to see all branches, including remote branches)

git checkout -b your-branch-name (to create a new branch and switch to it)

git switch -c branch_name (to create and switch to a new branch using the newer command)

git switch branch_name (to switch to an existing branch using the newer command)

git push -u origin branch_name (to push the new branch to the remote repository and set the upstream tracking)

git merge branch_name (to merge changes from another branch into the current branch)

git branch -d branch_name (to delete a branch after merging)
